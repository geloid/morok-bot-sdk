# Начало работы

Пошаговое руководство по созданию первого Морок-бота. Скриншоты панели разработчика, файл `.morokbot`, установка SDK, первый эхо-обработчик.

English version: [getting-started.md](./getting-started.md).

## 1. Включите режим разработчика

В приложении Морок: **Настройки -> О Мороке**, включите переключатель **Режим разработчика**. В настройках появляется одноименный раздел.

![Шаг 1: включение режима разработчика](./screenshots/step-1-developer-mode.png)

## 2. Откройте вкладку "Режим разработчика"

**Настройки -> Режим разработчика**. Пусто при первом заходе, одна кнопка: **Создать бота**.

![Шаг 2: вкладка Режим разработчика](./screenshots/step-2-bots-tab.png)

Каждый владелец может завести до 10 ботов. Удаленный бот сразу освобождает слот.

## 3. Шаг 1 окна создания: Описание

Аватар (картинка или юникод-символ), имя (до 64 символов), псевдоним (5-24 символа из `[a-zA-Z0-9-]`), описание (до 250 символов) и переключатель **Приватный бот**. К псевдониму автоматически добавляется суффикс `-bot`. Имя, описание и аватар можно изменить позже, псевдоним и приватность задаются один раз при создании.

![Шаг 3: шаг 1 окна создания](./screenshots/step-3-wizard-info.png)

Если получившийся псевдоним уже занят, окно создания скажет сразу. Приватный бот не показывается в поиске тем, кто его не запускал.

## 4. Шаг 2 окна создания: Внешний вид

Оформление чата с ботом: акцентный цвет, обои и фоновый паттерн. Те же настройки, что у бесед и каналов.

![Шаг 4: шаг 2 окна создания](./screenshots/step-4-wizard-appearance.png)

## 5. Шаг 3 окна создания: Управление

Этот шаг делает три вещи:

1. **Токен**. Окно создания генерирует и показывает токен сразу, ровно один раз, скопируйте его кнопкой-иконкой. Скачать его целиком как `.morokbot` (шаг 6) можно только после того, как вы нажмете **Создать бота** в самом конце: тогда токен закрепляется за ботом и появляется возможность скачать файл и перевыпустить токен.
2. **Команды**. До 32 слэш-команд (например `/help`, `/start`). Имя команды: строчные латинские буквы, цифры и подчеркивание, начинается с буквы, до 32 символов. Описания видны пользователю в списке команд бота.
3. **Разрешения**. Два переключателя: "Можно добавлять в беседы" и "Можно добавлять в каналы". Оба выключены по умолчанию.

![Шаг 5: шаг 3 окна создания](./screenshots/step-5-wizard-token.png)

Токен показывается ровно один раз. Если потеряли, перевыпустите его в редактировании бота.

## 6. Скачайте файл .morokbot

Кнопка **Скачать .morokbot** появляется только после нажатия **Создать бота** в самом конце окна создания, рядом с показанным токеном. Она объединяет токен со свежесгенерированным материалом Signal (identity-ключ, подписанный prekey, запас одноразовых prekey) в один JSON-файл.

![Шаг 6: скачивание .morokbot](./screenshots/step-6-download-morokbot.png)

В файле лежат приватные ключи бота. Ограничьте к нему доступ:

```bash
chmod 0600 ./bot.morokbot
echo '*.morokbot' >> .gitignore
echo 'bot-state/' >> .gitignore
```

В `examples/.gitignore` SDK обе строки уже есть.

## 7. Установите SDK

```bash
npm install morok-bot-sdk
```

Node 22+. Полный список требований в [README §Установка](../README.ru.md#установка).

## 8. Первый обработчик

```ts
import { MorokBot } from 'morok-bot-sdk'

const bot = await MorokBot.fromFile({ tokenFile: './bot.morokbot' })

bot.on('message', async (m) => {
    if (m.text) await bot.reply(m, { text: `эхо: ${m.text}` })
})

await bot.start()
console.log('бот поднят')
```

Запустите: `node --experimental-strip-types index.ts` (или после `npm run build`: `node dist/index.js`). Первый запуск создает `bot-state/` рядом с кодом.

## 9. Нажмите "Запустить" в чате с ботом

Откройте чат с ботом. При первом заходе вместо поля ввода внизу видна кнопка **Запустить**.

Нажмите. SDK получает событие `bot.on('start', ...)`, поле ввода разблокируется, и с этого момента бот может писать пользователю напрямую. Отозвать согласие можно в профиле бота строкой **Остановить**.

## 10. Управление ботом потом

Во вкладке Режим разработчика у каждого бота два действия:

| Действие      | Что делает                                                                                      |
|---------------|--------------------------------------------------------------------------------------------------|
| Редактировать | Все, кроме псевдонима и приватности: имя, описание, аватар, внешний вид, команды, разрешения.    |
| Удалить       | Мягкое удаление. Псевдоним освобождается сразу, все собеседники получают `user_deleted`.         |

Внутри Редактировать отдельная кнопка **Перевыпустить** токен. Используйте, если токен утек. Поведение:

- Сервер отзывает старый токен и закрывает все живые сессии.
- Окно создания показывает новый токен один раз.
- Подмените поле `token` в `.morokbot`, остальное (identity, prekeys) остается прежним.
- Перезапустите SDK.

Если потерян и ключевой материал, бота придется удалить и создать заново. Удаление убирает аккаунт бота целиком: у собеседников он превращается в удаленный аккаунт и пропадает из поиска. Новый бот будет отдельным аккаунтом, даже под тем же псевдонимом, согласие на него не переносится, поэтому каждому собеседнику придется заново найти бота и нажать "Запустить".
